# File Manager

智能文件整理 skill。工作方式：先诊断、再提案、等确认、最后执行。

## 核心原则

- **不替用户做决定**：建议，但最终结构由用户拍板
- **安全优先**：移动前确认、删除前备份
- **分步可逆**：每一步独立，出问题可以回滚
- **并行审计**：大规模整理时派子 agent 并行核查，保护主上下文

## 工作流程

### 阶段一：了解范围

先问清楚三件事（根据用户的初始描述，只问缺失的）：

1. **整理范围**：整个目录、指定子目录、还是某个项目？
2. **整理深度**：只做大类归拢，还是精细到每个文件？
3. **是否只出方案**：只出方案不执行，还是方案通过后直接干？

### 阶段二：扫描诊断

扫描目标目录。诊断报告格式：

```
## 扫描诊断：<目录路径>

### 当前结构
<目录树，只到2-3层>

### 发现的问题
| 类型 | 位置 | 说明 | 风险 |
|------|------|------|------|
| 散落文件 | /path | .py .png .docx 混在一起 | 低 |
| 重复项目 | /path/A, /path/B | 两个目录内容相同 | 中 |
| 大文件 | /path/file.tar.gz | 84MB 放在根目录 | 低 |
| 空目录 | /path/empty/ | 无内容 | 低 |
| 路径错误 | config.toml:24 | 引用了不存在的路径 | 严重 |
| 过期副本 | /path/old/ | 比主版本旧 | 中 |

### 磁盘占用 TOP 5
<最大的 5 个文件/目录>

### 重复文件
| MD5 | 路径 A | 路径 B | 大小 | 建议 |
|-----|--------|--------|------|------|
| a1b2c3... | /path/file1.tar.gz | /path/backup/file1.tar.gz | 84MB | 保留更新的 |
```

**找重复命令**：

```bash
# 精确重复（MD5 相同）
find <目录> -type f -exec md5sum {} \; | sort | uniq -w32 -d --all-repeated=separate

# 同名文件
find <目录> -type f -printf '%f\n' | sort | uniq -d

# 相同大小的文件（重复嫌疑）
find <目录> -type f -printf '%s %p\n' | sort -n | uniq -D -w8
```

原则：保留更新、命名更好的版本。**删除重复前必须确认。**

### 阶段三：设计提案

输出目标结构和变更清单：

```
## 整理方案

### 目标结构
<after 的目录树>

### 变更清单
| # | 操作 | 源 → 目标 | 风险 | 可逆 |
|---|------|----------|------|------|
| 1 | 移动 | /a/b.tar.gz → /archives/inbox/ | 低 | 是 |
| 2 | 删除重复 | /x (保留 /y) | 中 | 需确认 |

### 无法自动处理的事项
<需要用户手动决定的>
```

展示给用户，等反馈。用户可以：
- 直接批准 → 进入阶段四
- 修改部分项 → 更新方案后再确认
- 只保留方案 → 结束

### 阶段四：执行

按变更清单从低风险到高风险逐步执行。每完成一步汇报。

### 阶段五：路径修正

变更涉及路径移动时，扫描以下位置更新引用：

1. 项目配置（CLAUDE.md, settings.json, settings.local.json）
2. 定时任务（crontab, systemd timer）
3. 脚本（所有 .sh/.py）
4. 服务配置（/etc/systemd/system/, docker-compose.yml）
5. hooks 配置
6. 文档和 README

用 `grep -r "旧路径"` 搜索，逐处修正。每改一处确认目标路径确实存在。

### 阶段六：派子 agent 审计

修正完后，派 2-3 个子 agent 并行核查：
- 子 agent 1：定时任务 + 系统服务
- 子 agent 2：项目文件完整性 + 路径引用
- 子 agent 3：配置文件 + hooks + 文档

收集所有审计报告，修复发现的问题。

### 阶段七：最终报告

```
## 整理完成报告

### 整理前后对比
- 目录数：12 → 5
- 根目录文件：8 → 2
- 磁盘释放：XXX MB

### 已修正路径引用
| 文件 | 旧路径 | 新路径 |
|------|--------|--------|
| ... | ... | ... |

### 审计结果
- ✅ 定时任务全部正常
- ✅ 项目文件完整
- ⚠️ X个问题（已修复）
```

### 可选：项目生命周期标注

对项目目录做健康检查：

| 标签 | 含义 |
|------|------|
| 🔥 活跃 | 正在开发 |
| ⏸️ 暂停 | 暂时搁置 |
| 📦 归档 | 已完成 |

## 安全红线

### 绝对不碰

以下目录和文件，无论什么情况都不能移动、删除、重命名：

```
~/.ssh/              ← SSH 密钥
~/.gnupg/            ← GPG 密钥
~/.claude/           ← Claude Code 配置（可以改内部文件内容，不能动目录结构）
~/.bashrc            ← Shell 配置
~/.profile           ← 登录配置
~/.gitconfig         ← Git 全局配置
/etc/                ← 系统配置（只读）
/var/log/            ← 系统日志（只读）
```

### 需要用户明确确认

- 删除任何文件（包括 MD5 完全相同的重复文件——必须单项确认）
- `rm -rf` 任何目录
- 修改系统服务文件
- 修改 crontab
- 重启 systemd 服务
- 移动正在被服务引用的目录

### 移动前检查

1. **目标父目录存在**：先建好
2. **不覆盖已有文件**：目标已存在同名文件 → 停下来问
3. **不在使用中**：grep crontab、systemd service 确认没有引用源路径

### 回滚原则

- 移动操作 → `mv 目标 源` 回滚
- 删除操作 → 执行前 `cp -a` 备份到 `/tmp/`
- 路径修改 → 记录改前内容，可以原样改回
